# Commit generated platform variants, rebuilt by CI

Platform variants (`plugins/maister-copilot/`, `plugins/maister-pi/`) are generated by `platforms/<platform>/build.sh` from the single source plugin `plugins/maister/`, yet they are committed to git rather than gitignored. We chose this so users can install a variant straight from a checkout (the Claude Code marketplace serves `maister-copilot` directly; Pi's `install.sh` copies from the built directory) without running a build. The drift risk this creates is mitigated by CI (`build-copilot.yml`, and its Pi equivalent) rebuilding and auto-committing variants on every push touching `plugins/maister/**` or `platforms/**` — humans must never edit variant files by hand.

## Considered Options

- Gitignore variants, build on demand — cleanest git history, but breaks install-from-checkout and the marketplace flow.
- Commit variants with manual rebuild discipline — the previous state; caused the v2.1.8/v2.2.1 Pi drift this decision fixes.
